01

Mapa do fluxo editorial

O portal separa texto (barato, rápido) de visual (imagem IA + HTML). Você sempre revisa a copy antes de gastar crédito de imagem.

Briefing → 3 propostas ou campanha (3–10) → Aprovação humana → Imagem + layout A–Q → Galeria + legenda
Propostas não escolhidas → Banco editorial → Gerar visual depois
Use Esc para fechar modais e overlays. Links diretos: index.html#arte=slug-da-arte reabre o modal da arte.
02

Galeria de artes

Para que serve: ser o ponto único de consulta e publicação — ver a arte, copiar legenda e abrir o editor sem navegar pastas no repositório.

  1. 1
    Clique no card — abre o modal com preview, metadados e legenda completa pronta para copiar.
  2. 2
    Hover no card — atalhos: Legenda · Editar · PNG.
  3. 3
    Abrir Editor — vai para artes/{slug}/arte.html com painel visual v3.
  4. 4
    Filtro backup — toggle “Mostrar testes / backup”. Para quê: esconder rascunhos de teste da vista principal; a galeria fica limpa para revisão e apresentação.

Modal de legenda

Para que serve: publicar no social em segundos — legenda completa, metadados e preview num só lugar.

Badge do tipo, headline, subtítulo, palavras azuis, cidade, layout, formato e botão copiar legenda.

03

Pedido de post e calendário

Para que serve: iniciar conteúdo novo sob demanda — com briefing livre ou seguindo o ritmo editorial da semana.

  1. 1
    Campo de briefing — descreva tema, tom, cidade, sugestão de fundo (MASP, Paulista…). Vazio = calendário automático.
  2. 2
    Botão ✦ — dispara POST /api/pedido e retorna 3 propostas de texto.
  3. 3
    Calendário editorial — barra Seg / Qua / Sex. Para quê: saber qual tipo de post publicar hoje sem abrir planilha ou temas.json; alinha time humano e automação.
Sem npm run dev, a galeria só exibe artes já publicadas — o painel de pedido não funciona.
04

Aprovação de propostas (3 rotas)

Para que serve: gate de qualidade antes da imagem — você valida copy, título e CTA enquanto ainda é barato mudar de ideia.

  1. 1
    Preview do título — mini-thumb com quebras e azul. Para quê: detectar título longo, quebra ruim ou palavra azul incorreta antes de gastar crédito de imagem.
  2. 2
    Edite o CTA visual — campo por peça (ex.: INSCRIÇÕES ABERTAS). Aparece nos layouts E, J, L, O, P.
  3. 3
    GERAR IMAGEM — radio na peça principal → gera visual agora.
  4. 4
    Guardar no banco — checkbox nas outras rotas → copy reservada para gerar depois.
  5. 5
    Sugestão de imagem — textarea opcional para forçar cenário visual na geração.

05

Modo campanha (3–10 peças)

Para que serve: produzir um lote editorial completo (semana de lançamento, push de patrocínio, etc.) numa única sessão, com variedade de tipo e formato.

Objetivos

Inscrições · Patrocinadores · Palestrantes · Agenda — cada um com sequência de tipos de post.

Formato por peça

Para que serve: cobrir feed, quadrado e stories na mesma campanha — sem planejar formato manualmente em cada peça.

Rotação: Feed Vertical → Quadrado → Stories → Vertical… Cada card mostra o formato planejado.

Pack ZIP

Para que serve: handoff para mídia/agência — legendas e copy organizadas por pasta, prontas para agendar fora do portal.

Botão ⬇ Pack ZIP exporta copy, legendas e headlines — inclui PNG/HTML se já gerados.

  1. 1
    Abra Modo campanha no painel de pedido.
  2. 2
    Escolha objetivo e quantidade (3–10).
  3. 3
    Revise no overlay — mesma UI de propostas, com badge de formato.
  4. 4
    Baixe o pack antes de aprovar, se quiser enviar copy ao time de mídia.
06

Editor visual v3

Para que serve: polimento final sem regerar IA — reposicionar foto, título e CTA até a arte ficar publicável.

Painel Fundo

Posição, zoom, opacidade, espelhar, saturação, overlay e cor do canvas.

Painel Elementos

Logo, título, subtítulo, CTA pill e ecossistema — alinhamento, escala e deslocamento.

Contador headline

Para que serve: evitar exportar arte com título inválido — você vê palavras, linhas e palavras azuis enquanto ajusta o layout.

No painel Título: contador em tempo real + aviso se estourar as regras editoriais.

Formato de export

Para que serve: gerar PNG no tamanho certo para cada rede (feed 4:5, quadrado, stories) a partir da mesma arte base.

Seletor no topo: Vertical 4:5 · Quadrado 1:1 · Stories 9:16.

💾 Salvar

Persiste estado do editor e regenera thumb (requer servidor local).

⬇ Exportar PNG

Export na resolução do formato escolhido (1080×1350, 1080×1080 ou 1080×1920).

Templates A–Q em galeria-templates usam o mesmo editor — salvar um template promove o padrão do layout.
